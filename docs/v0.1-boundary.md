# v0.1 Boundary: Compendium-as-Code

This repository is an executable v0.1 packet for **compendium-backed source ingestion as code**.

## Scope

- Represent source manifests, source-lineage records, importer contracts, and evidence of transformation as reviewable source.
- Provide minimal domain patterns for moving upstream Compendium data into governed outputs.
- Keep licensing, attribution, and provenance explicit and auditable.

## Non-canon posture

This repository is `seed`/`v0.1-draft` only until explicitly adopted by HUMMBL authority.

## Boundaries

- Do not treat import logic as policy or enforcement policy.
- Do not vendor full upstream tables or sensitive upstream assets without explicit license review.
- Do not define final governance outcomes from this repository.
- Do not claim compliance outcomes without explicit review and exception receipts.

## Initial practical user

Engineering, taxonomy, and data teams who need:

- source identity and retrieval receipts,
- transformation traceability,
- importer validation evidence,
- auditable compendium packet updates.
